Projects

Web – Manual and Automation QA

I carried out the quality assurance of an already developed web application, working as part of a team under Scrum agile methodologies. Work management was carried out through weekly sprints using Jira, actively participating in the Sprint Planning, dailys, and Sprint Review ceremonies at the end of each sprint.

I designed the tests for the user stories following the BDD methodology, based on the defined acceptance criteria. Scenarios were created in Gherkin language to provide tests that were readable and understandable for the entire team, managing them with AssertThat to maintain traceability between requirements, tests, and results within Jira. The tests were executed per user story and the results were recorded in both web and mobile environments.

During each sprint, functional testing was carried out and, at the end of the sprint, exploratory testing was performed to identify defects not covered by the defined test cases. The detected issues were reported in Jira, linking them to the corresponding user story to ensure proper traceability between requirements, tests, and defects.

At the end of each sprint, the release was prepared in Jira and the information related to the delivery was documented in Confluence, including the complete documentation of the test plan, execution results, and the product’s quality status.

After the release, the application's main flows were identified and regression tests were automated using Playwright and Pytest with Python. These tests were integrated into a continuous integration workflow with GitHub Actions, configured to automatically run the regression on a weekly basis, aligned with the end of each sprint.

QA Portfolio – Web Developer and QA in API Testing

I developed a web portfolio using HTML5, CSS3, and JavaScript, integrating a form connected to an API generated with Supabase. Database tables were designed, and SQL queries were performed for the insertion and retrieval of form test data.

I conducted API testing using Postman, creating tests for GET (retrieval), POST (creation), PATCH (editing), and DELETE (deletion) requests. Tests were implemented to validate positive, negative, and authentication response codes, as well as the correct structure and content of the responses.

Additionally, a continuous integration workflow was implemented with GitHub Actions, configuring Newman to automatically execute the tests created in Postman on every repository change.
